# methods/

Operational, project-internal methodology specs. One sub-folder per
method (`methods/<method-slug>/`); each carries a required `rule.md`
and any adjuncts (codebook PDFs, transparency CSVs, notebooks tying
the rule to its diagnostic outputs).

This folder is for *what we decided to compute and why*, paired with
diagnostic counts that prove the rule is in force. Peer-reviewable
methodology calls (deflator choice, identification strategy) live in
`decisions/`; distilled domain claims with citations live in
`wiki/concepts/`.

Full protocol: `.claude/conventions/methods.md` (read on demand).
Worked example: `EXAMPLE_method/rule.md` (delete or replace once
real methods are documented).
