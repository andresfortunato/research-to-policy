# data_sources/

Reference docs for every external data API, dataset, or codebook this
project consumes. One file per source-and-purpose; flat folder; the
`INDEX.md` here is the quick-nav.

This folder holds **how to access** external data — endpoints, query
shapes, parsing recipes, pitfalls. The data itself lives in `data/`;
peer-reviewable methodology calls live in `decisions/`; distilled
domain claims live in `wiki/concepts/`.

Full protocol: `.claude/conventions/data-sources.md` (read on demand).
Worked example: `EXAMPLE_world_bank_api.md` (delete or replace once
real sources are documented).
