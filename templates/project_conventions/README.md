# project_conventions/

Project-bespoke style and process rules — visualization, writing
voice, slide design, naming, and similar recurring decisions
this engagement makes locally. One file per domain; flat folder;
the `INDEX.md` here is the quick-nav.

This folder holds **decisions this project made about how it
does work** — not framework-shared protocols (those are in
`.claude/conventions/`), not external-system reference docs
(those are `data_sources/`), not operational compute rules
(those are `methods/`).

Full protocol: `.claude/conventions/project-conventions.md`
(read on demand). Worked example: `EXAMPLE_visualization.md`
(delete or replace once real conventions are documented).
